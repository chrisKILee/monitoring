# PRD - Product Requirements Document

> **Why**: Claude.ai 계정을 여러 개 관리하면서 만료일을 놓쳐 서비스가 끊기는 문제를 방지하고, 사용량 초과를 사전에 예측하여 항상 안정적으로 Claude를 활용하기 위해

## 작성일
2026-04-10

## 1. 문제 정의 (Problem Statement)

6~20개의 Claude.ai 계정(기관 ID 단위)을 운영하면서 다음 문제가 발생:

- **만료일 누락**: 각 계정의 구독 만료일을 수동으로 확인해야 하며, 놓치면 갑작스러운 서비스 중단
- **사용량 파악 불가**: 현재 사용 속도로 언제 쿼터가 소진될지 알 수 없음
- **다중 계정 일괄 관리 불가**: claude.ai 웹에서는 계정별로 일일이 접속해야 함

## 2. 목표 (Goals)

- [ ] 5시간마다 자동으로 전체 계정 사용량 수집
- [ ] 만료 7일 이내 Google Chat 알람 발송
- [ ] 현재 사용 속도 기준 5h/7day 내 쿼터 초과 예측 및 알람
- [ ] 대시보드에서 전체 계정 상태 한눈에 파악
- [ ] 세션 쿠키 갱신을 최소한의 수작업으로 처리 (콘솔 스크립트 제공)

## 3. 비목표 (Non-Goals)

MVP에서 제외되는 항목:
- 자동 구독 갱신 (결제 자동화)
- 이메일 알람 (Google Chat만 지원)
- 다른 AI 서비스 모니터링 (Claude.ai 전용)
- 모바일 앱
- 다중 사용자 지원 (개인용 단일 사용자)
- 세션 쿠키 완전 자동 갱신 (브라우저 자동화)

## 4. 사용자 페르소나 (User Personas)

### 페르소나 1: 계정 관리자 (나)
- **특징**: Claude.ai 계정 6~20개를 직접 관리하는 개인 개발자
- **주요 니즈**:
  - 만료 전 충분한 사전 경고 (7일 이상)
  - 사용량이 위험 수준인지 한눈에 파악
  - 쿠키 갱신 작업을 최소화하고 싶음
  - 대시보드를 자주 확인하지 않아도 알람으로 파악 가능

## 5. 핵심 기능 (Features)

| 기능 | 우선순위 | MVP 포함 | 설명 |
|------|----------|----------|------|
| Vercel Cron 5h 주기 수집 | P0 | ✅ | `0 */5 * * *` 스케줄로 전체 계정 API 조회 |
| PostgreSQL 사용량 저장 | P0 | ✅ | 조회 결과 히스토리 보존, 추세 분석 가능 |
| 다중 계정 관리 (6~20개) | P0 | ✅ | 계정별 기관 ID + 세션 쿠키 DB 암호화 저장 |
| 대시보드 시각화 | P0 | ✅ | 계정별 사용량/만료일 카드 + 추세 차트 |
| 5h/7day 초과 예측 | P0 | ✅ | 현재 사용 속도로 쿼터 소진 시점 계산 |
| 만료일 사전 경고 알람 | P0 | ✅ | 7일 이내 만료 시 Google Chat 발송 |
| Google Chat Webhook 알람 | P0 | ✅ | 만료/초과 예측 시 실시간 알람 |
| 브라우저 콘솔 쿠키 추출 스크립트 | P0 | ✅ | 수작업 최소화, 복붙으로 쿠키 갱신 |
| 계정 관리 UI | P0 | ✅ | 계정 추가/수정/삭제, 쿠키 갱신 인터페이스 |
| API 조회 실패 알람 | P1 | ✅ | 쿠키 만료 등으로 조회 실패 시 Google Chat 알람 |

## 6. 성공 지표 (Success Metrics)

- [ ] 만료일을 놓쳐 서비스가 중단되는 일 0건
- [ ] 쿼터 초과 예측 정확도 90% 이상 (5h 기준)
- [ ] 전체 계정 조회 완료 시간 5분 이내
- [ ] 대시보드 로딩 속도 2초 이내
- [ ] 쿠키 갱신 작업 시간 계정당 1분 이내

## 7. 제약사항 (Constraints)

- **기간**: 1주일 이내 배포
- **스택**: Next.js 15 App Router + Prisma + PostgreSQL + Vercel 고정
- **인증**: claude.ai 세션 쿠키 기반 (공식 API 없음, 브라우저 세션 활용)
- **비용**: Vercel Free/Pro + Supabase/Neon Free Tier 범위 내
- **보안**: 쿠키 평문 저장 금지 (AES-256 암호화 필수)

## 8. 열린 질문 (Open Questions)

- [ ] claude.ai API 응답에서 만료일 필드 이름 확인 필요 (첫 조회 후 확인)
- [ ] Vercel Cron Free Tier 제한 확인 (월 2회 → Pro 필요 여부)
- [ ] 세션 쿠키 유효 기간 파악 (얼마나 자주 갱신해야 하는가)
- [ ] 암호화 키 관리 방식 결정 (Vercel 환경변수 vs 외부 KMS)
